## 项目简介

本项目旨在模拟 PyTorch 的自动求导功能，使用 CUDA 实现高效的张量（Tensor）计算和自动梯度传播。项目支持在 GPU 和 CPU 上运行，提供了基本的算子（如 ReLU 和 Sigmoid）以及自动反向传播功能。

---

## 功能实现

### 1. Tensor 的实现

`Tensor` 是项目的核心数据结构，支持以下功能：
- **数据存储**：使用 `std::shared_ptr` 管理张量数据和梯度，支持动态分配和释放内存。
- **设备支持**：支持 `CPU` 和 `GPU` 两种设备，使用 `cudaMalloc` 和 `cudaMemcpy` 实现 GPU 数据管理。
- **自动求导**：每个 `Tensor` 可以记录其创建时的算子（`Function`），通过调用 `backward()` 方法实现反向传播。
- **数据转换**：提供 `fromVector` 和 `dataVec` 方法，用于在标准向量和张量之间进行数据转换。

#### Tensor 的核心成员：
```cpp
std::shared_ptr<dtype[]> data, grad; // 数据和梯度存储
device_t device;                     // 设备类型（CPU 或 GPU）
std::shared_ptr<Function<dtype>> creator; // 创建张量的算子
```

---

### 2. Function 的实现

`Function` 是所有算子的基类，支持以下功能：
- **前向传播**：通过 `forward` 方法实现算子的计算逻辑。
- **反向传播**：通过 `backward` 方法实现梯度的反向传播。
- **依赖管理**：通过 `parents` 方法记录输入张量，支持动态计算图。

#### 已实现的算子：
- **ReLU**：实现了 ReLU 激活函数及其反向传播。
- **Sigmoid**：实现了 Sigmoid 激活函数及其反向传播。

---

### 3. 自动求导机制

自动求导通过以下步骤实现：
1. **记录依赖**：每个 `Tensor` 在创建时记录其创建算子（`creator`）。
2. **拓扑排序**：调用 `backward()` 时，首先对计算图进行拓扑排序，确保梯度按依赖顺序传播。
3. **反向传播**：依次调用每个算子的 `backward` 方法，计算并累积梯度。

## 正确性验证

### 1. 单元测试
- 每个算子（如 ReLU 和 Sigmoid）都通过单元测试验证其前向和反向传播的正确性。
- 测试用例包括：
  - CPU 和 GPU 上的计算结果是否一致。
  - 梯度传播是否正确。

### 2. 数值梯度验证
- 使用数值梯度（finite difference）验证自动求导的正确性：
  - 对输入张量的每个元素，计算数值梯度。
  - 比较数值梯度和自动求导的梯度，确保误差在容许范围内。

### 3. 示例程序
- 提供了一个示例程序（`example.cu`），测试了以下功能：
  - 张量的创建与设备转换。
  - ReLU 和 Sigmoid 的前向传播与反向传播。
  - 复杂计算图的梯度传播。
